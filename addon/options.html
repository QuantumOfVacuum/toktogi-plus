Change log

0.6.6

  • Adds design updates from Firefox 0.6.4 and 0.6.5 to Chrome extension
  • Now displays Hanja (Chinese characters) for definitions that include them (~38k words out of 117k total words in the dictionary)

0.6.5 (Firefox)

  • Fixes issue with toktogi css styling non-toktogi elements on pages
  • Fixes issue with writing "Toktogi is on" in input boxes on some pages in Firefox

0.6.4 (Firefox)

  • Fixes missing toolbar button in Firefox 40 (had to rush to get this fix out, so some designs/features are not complete)
  • Adds lock feature to definition box
  • Adds some material design, finished design will be in the next update

0.5.1 (Firefox)

  • Adds support for using Toktogi on local html files instead of only http/https

0.5 (Firefox)

  • Dictionary now loads and unloads on enable/disable to save memory when not in use

0.4.1 (Chrome)

  • Adds slight delay before popping up the definition box
  • Adds support for using Toktogi on local html files instead of only http/https

0.4

  • Adds recognition for conjugated forms of verbs
  • Adds getting started guide

0.3

  • Fixes bug where looking up words adjacent to previously looked up words wouldn't work
  • Now shows multiple definitions for words if available
  • Definitions box now always shows up underneath the highlighted word
  • Injects script at a more optimal time during page loading process
